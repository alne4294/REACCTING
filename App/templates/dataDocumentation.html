About the Data Used for this App

 

The sourcecode for the app can be found on Github.  Since it is a private repository, you will need to be requested to be added as a collaborator.

Issues with this app can be submitted to the issue tracker at the Github repository (preferred method) or emailed to Alexia Newgord at alexia@alexianewgord.com.

When replacing or adding to any of the datasets, it is critical to maintain the same relative paths. In other words, put the new file(s) in the same location as the old file(s) so that the application knows where to find it.

 

PEMs

Original format: Matlab

Steps to import:

  1. Pull the needed files from box.net
  2. Convert using Convert_Mat_2_CSV.m, which creates Matlab and CSV copies with the correct date formatting
  3. Import into MongoDB with import_pems.rb (which is ran using run_import_pems.sh)
  4. To share this database, replace the existing compressed database with a new .bson compression. Details on how to use mongodump to do this can be found here.

 

Household Coordinates

Original format: CSV

Steps to import:

  1. Copy of file currently exists in repository as householdsamplelatlong.csv
  2. Convert to JSON using the JSON to CSV converter.  All values should be wrapped in double quotes and the new file should be named householdData.json

 

PEMs Comparison Info

Original format: XLSX

Steps to import:

  1. None.  However the location, formatting, and name of this file must stay the same to be correctly parsed (ComparingAcrossStovesAndSeasons.xlsx)

 

GPS Data

Original format: CSV

Steps to import:

  1. Get the CSV files from Dropbox
  2. Hand clean any clearly duplicated data entries or entries that seem really off (for example, records in Boulder CO or with other clearly incorrect longs and lats).  The app with do further cleaning, but it is best not to send an excessive amount of invalid data over the network.
  3. Convert to JSON using the JSON to CSV converter. All fields should be wrapped in double quotes and the filename should be Phone1.json or Phone2.json respectively. Note: the app currently only supports two phones.
